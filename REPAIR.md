# REPAIR.md - 记忆修复流程

## 触发条件

- 使用者指出错误（即时触发）
- 重复犯同类错误
- 周复盘跌入C/F区

## 修复三层优先级

```
第一层：事前预防失效
→ 检查：任务前是否读了ERRORS.md活跃区？
→ 修复：在agent的SOUL.md里加入"任务前读ERRORS.md"约束
→ 输出：在修复报告里标注"已加入SOUL.md"

第二层：清单描述不够具体
→ 检查："检查方式"能否写出可操作的自验步骤？
→ 修复：重写该条记录直到"检查方式"能写出具体动作为止
→ 验证：下次同类任务未再犯 → 修复有效
         再犯 → 描述仍不够具体，扣分加重+继续重写

第三层：规范文件本身有问题
→ 检查：SOUL.md/TOOLS.md里是否有歧义或矛盾？
→ 修复：与使用者一起审查，找到模糊点修改
→ 此层需要使用者介入，agent不能单独完成
```
