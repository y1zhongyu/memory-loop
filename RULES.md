# RULES.md - 评分规则与犯错边界

## 基础规则

- 每周从100分开始，只扣不加，每周重置
- 上周分数归档，用于观察趋势

## 分数区间

| 分数 | 评级 | 含义 |
|------|------|------|
| 90-100 | A · 稳定可靠 | 极少失误，记忆机制正常 |
| 70-89 | B · 基本合格 | 有错误，需关注重复趋势 |
| 50-69 | C · 需要改进 | 重复错误明显，记忆力存疑 |
| 低于50 | F · 触发重置 | 强制执行全面诊断 |

## 扣分标准

| 错误类型 | 首次 | 重复犯 |
|---------|------|--------|
| 信息遗漏 | -3 | -6 |
| 事实错误 | -5 | -10 |
| 格式违规 | -2 | -4 |
| 流程跳步 | -4 | -8 |
| 沟通缺失 | -3 | -6 |

**触发方式按信号强度分级：**

✅ 强信号（立即触发，当轮执行，不等再催）：
- 用户明确指出错误（"这错了"、"不对"、"不是这个意思"）
- 用户要求重来/推翻输出（"重新做"、"算了我来"）
- 用户直接忽略输出重新发需求（沉默式否定）
→ 首次：正常扣分 / 重复犯：×2

⚠️ 弱信号（记为"待确认"，当轮主动问用户，有回答再决定）：
- 用户表达迟疑（"这不太对吧"、"感觉哪里不对"）
- 用户保留式接受（"好吧"、"凑合"、"随便"）
- 用户修改AI内容后直接用（说明AI没做到位）
→ 用户确认是错误 → 触发扣分
→ 用户说是改需求 → 不触发

批量错误规则：
- 用户说"这几点都有问题"→ 逐条列出+逐条确认+逐条分别记录
- 禁止合并为一条（合并会导致后续单独犯某项时无法触发"重复犯"）

## 犯错边界

✅ 算犯错：
- 违反规范文件明文规定
- 输出可核查的错误事实
- 遗漏流程必经步骤
- 使用者明确指出

❌ 不算犯错：
- 无明确标准的"可以更好"
- 风格语气等主观偏好
- 判断类决策（无唯一正确答案）

## 触发机制

| 优先级 | 触发形式 | 处理方式 |
|--------|---------|---------|
| 即时强制 | 强信号（明确指出/要求重来/沉默否定） | 当轮立即记录+扣分，不等下一轮，不等再催 |
| 确认后触发 | 弱信号（迟疑/保留/修改后用） | 当轮主动确认，用户点头后记录+扣分 |
| 不触发 | 用户改需求/风格主观偏好 | 不记录，正常执行 |

| 时机 | 触发内容 |
|------|---------| 
| 强信号识别 | 当轮立即：记录+扣分+触发REPAIR.md+阈值报警 |
| 弱信号识别 | 当轮主动确认，有回答再决定 |
| 周一（周复盘） | 汇报本周原始扣分总计+生成周报 |
| 新的一周（周一） | heartbeat触发重置+归档 |
