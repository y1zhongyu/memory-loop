# RULES.md - 评分规则与犯错边界

## 基础规则

- 每周从100分开始，只扣不加，每周重置
- 上周分数归档，用于观察趋势

## 分数区间

| 分数 | 评级 | 含义 |
|------|------|------|
| 90-100 | A · 稳定可靠 | 极少失误，记忆机制正常 |
| 70-89 | B · 基本合格 | 有错误，需关注重复趋势 |
| 50-69 | C · 需要改进 | 重复错误明显，记忆力存疑 |
| 低于50 | F · 触发重置 | 强制执行全面诊断 |

## 扣分标准

| 错误类型 | 首次 | 重复犯 |
|---------|------|--------|
| 信息遗漏 | -3 | -6 |
| 事实错误 | -5 | -10 |
| 格式违规 | -2 | -4 |
| 流程跳步 | -4 | -8 |
| 沟通缺失 | -3 | -6 |

**唯一触发方式：使用者指出错误**
- 首次：正常扣分
- 重复犯：×2

## 犯错边界

✅ 算犯错：
- 违反规范文件明文规定
- 输出可核查的错误事实
- 遗漏流程必经步骤
- 使用者明确指出

❌ 不算犯错：
- 无明确标准的"可以更好"
- 风格语气等主观偏好
- 判断类决策（无唯一正确答案）

## 触发机制

| 时机 | 触发内容 |
|------|---------|
| 使用者指出错误 | 记录+扣分，触发REPAIR.md |
| 分数低于70 | heartbeat汇报错误清单+问题模式 |
| 使用者要求复盘 | 执行DIAGNOSIS.md |
| 周复盘C/F区 | 执行DIAGNOSIS.md全面诊断 |
| 新的一周（周一） | heartbeat触发重置+归档 |
