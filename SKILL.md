# memory-loop

## 核心价值：异步审计与认知沙盘

1. **白天 0 干扰**：彻底剥离运行时的计分逻辑，将响应延迟降至最低。
2. **异步审计**：通过晚间脚本回溯日志，实现精准结账与深度复盘。
3. **沙盘实验室**：通过 `DECISIONS.md` 提供非约束性推演空间。
4. **记忆新陈代谢**：周级清理冗余信息，保持灵魂精简。

## 文件结构

```
memory-loop/
├── SKILL.md        # 异步审计协议与日常规范
├── RULES.md        # 评分标准与标记规则 (异步版)
├── ERRORS.md       # 错误清单（由审计脚本维护）
├── SCOREBOARD.md   # 分数看板（由审计脚本维护）
├── DIAGNOSIS.md    # 诊断模板（含日报/周报标准格式）
├── REPAIR.md       # 记忆修复流程 (高风险动作)
├── DECISIONS.md    # 认知沙盘（讨论草稿/转正缓冲区）
└── scripts/
    └── audit_daily.py  # 晚间审计核心脚本
```

## 日常运行协议 (白天 0 干扰模式)

### 1. 初始化加载 (Session 启动)
- **指令**：Session 开始后的第一轮回复，必须执行一次 `read SKILL.md` (摘要) 和 `read ERRORS.md` (活跃区清单)。
- **目的**：将底线规则载入当前活跃记忆，后续对话不再重复读取。

### 2. 异步挂账 (发现错误时)
- **严禁动作**：白天的任务执行过程中，严禁执行 `write/edit` 修改 `ERRORS.md` 或 `SCOREBOARD.md`。
- **唯一操作**：仅在当轮回复末尾打标：`[Err: #ID (简述)]`。
- **自省原则**：若发现用户产生不满（纠错、重做、忽略输出），必须主动打标。未打标被脚本扫出视为“逃单”，加倍扣分。

### 3. 沙盘实验室协议 (DECISIONS.md)
- **定位**：非约束性讨论草稿纸。
- **场景**：方案 A/B 对比、复杂逻辑推演、暂不确定的格式调整。
- **计分豁免**：在 `## [推演]` 区块内的内容不计入审计扣分。
- **转正流程**：
  - 用户确认：“就按这个办” / “正式存入规范”。
  - 动作：`read DECISIONS -> write SOUL.md/模板 -> 清空 DECISIONS`。

## 审计与汇报协议

### 1. 晚间审计 (每日 20:00)
- **物理触发**：通过系统级 `crontab` 强制执行 `audit_daily.py`。
- **逻辑**：扫描当日所有 Session 日志 -> 抓取 `[Err]` 标记 -> 执行计分 -> 推送《今日记忆健康日报》。
- **日报标准模板**：
    1. **结账明细**：[Err: #ID] 标题、证据回溯、根因分析、修正建议。
    2. **算力消耗分布**：分类展示各业务标签的 Token 消耗与占比。
    3. **沙盘待处理项**：列出 `DECISIONS.md` 中的推演项。
    4. **逃单核查**：脚本自动交叉核对用户负面信号与 AI 标记。
- **三层验证协议**：所有修正建议必须符合“根因-动作-副作用评估”三要素，严禁敷衍建议（少于20字或无副作用评估视为违规）。

### 2. 全量复盘 (每周一 10:30)
- **职能**：
  - 生成《周报》，展示趋势与系统级对策（含深度病灶诊断模块）。
  - **记忆新陈代谢**：扫描 `MEMORY.md` 提取存疑（🟡/❓标记）、过时（>60天未用）及冗余信息，拉出“清理确认单”。
  - **周级瘦身计划**：归档 30 天未犯错误，分数重置为 100。

## 修复回血机制

- **唯一途径**：完整执行 `DIAGNOSIS.md` + `REPAIR.md`。
- **上限约束**：修复完成后分数最高恢复至 80 分，保持系统对二次犯错的敏感度。
